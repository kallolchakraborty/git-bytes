Git Internals Deep Dive

schedule3 min readsignal_cellular_4_barAdvanced / Staff+
staff+gitinternalsperformance

Explore Git's internal data structures — commit-graph, reachability bitmaps, and MIDX files — that power efficient operations at scale, and learn the same debugging techniques used by Git core contributors.

quick_reference Staff+ Cheat Sheetexpand_more

Core Commands

  • git cat-file --batch — bulk object inspection. Pipe object SHAs for efficient scanning.
  • git verify-pack .git/objects/pack/*.idx — validate packfile integrity.
  • git rev-list --objects --all — list all objects in the repo.
  • git gc --auto — auto-garbage-collect. Prefer over --aggressive.

Staff+ Patterns

  • Use git cat-file --batch-check in scripts to identify large objects: pipe git rev-list --objects --all | awk '{print $1}' to batch-check with object sizes.
  • Monitor git count-objects -v for loose object count. High loose count = GC needed. Automate GC in cron.

Watch Outs

  • git gc --aggressive can take hours on large repos and provides marginal benefit over git gc --auto.
  • Delta compression in packfiles is a read-time trade-off: deeper delta chains = smaller packs but slower object access.

Interview Angle

Google L6 asks about Git hosting design. Understanding the object model and Merkle DAG is essential. Know that the SHA-1 hash is computed as sha1(type + ' ' + size + '\0' + content) and that identical files always produce the same blob hash (automatic dedup).

Commit-Graph

The commit-graph file (.git/objects/info/commit-graph) stores commit metadata — parent IDs, generation numbers, commit timestamps — in a compressed binary format. It speeds up git log --graph, merge-base computation, and reachability queries by 10-100x. Enable via git commit-graph write. Git 2.18+ writes it automatically during git gc.

language-bash
# Write commit-graph for reachability speedup
git commit-graph write --reachable
# Writes .git/objects/info/commit-graph

# Write commit-graph for all refs
git commit-graph write --stdin-commits < <(git rev-list --all)

# Verify commit-graph integrity
git commit-graph verify

# Check commit-graph stats
git commit-graph read --show-format

# Enable commit-graph auto-writes
git config core.commitGraph true
git config gc.writeCommitGraph true

# Performance comparison
time git merge-base feature main
# Without commit-graph: ~12s for 500K commits
# With commit-graph: ~0.08s

Reachability Bitmaps

Bitmaps (.bitmap files paired with packfiles) track which objects are reachable from each ref using a bit array. One bit per object per ref. Git uses EWAH (Enhanced Word-Aligned Hybrid) compression for sparse bitmaps. Clone/fetch performance: instead of walking the entire DAG to enumerate objects, Git ORs bitmaps for the requested refs — O(1) object enumeration.

language-bash
# Bitmap index files (auto-generated during repack)
git repack -adb  # creates bitmap index
# Generates: .git/objects/pack/pack-<hash>.bitmap

# Check if bitmap exists
ls .git/objects/pack/*.bitmap

# Inspect pack with bitmap
git verify-pack -v .git/objects/pack/pack-*.idx | head -20

# Count bitmap-reachable objects
git rev-list --count --objects --all --use-bitmap-index
# vs without:
git rev-list --count --objects --all  # slower, no bitmap

# Force bitmap creation
git config pack.writeBitmapHashCache true
git config pack.writeReverseIndex true

MIDX Files

Multi-pack index (MIDX) files enable Git to query across multiple packfiles efficiently. Without MIDX, Git opens each idx file sequentially. MIDX creates a unified index (.git/objects/pack/multi-pack-index) that maps object IDs to packfile + offset. Essential for repositories with many small packfiles (frequent fetches, incremental repacks).

language-bash
# Create multi-pack index
git multi-pack-index write

# Verify MIDX integrity
git multi-pack-index verify

# Expire old packfiles (remove unreachable objects)
git multi-pack-index expire

# Repack using MIDX (consolidate packfiles)
git multi-pack-index repack --batch-size=2G

# MIDX + bitmap integration
git repack --write-midx --write-bitmap-index -d

# Check MIDX stats
git multi-pack-index --object-dir=.git/objects verify 2>&1

# Performance: query across packfiles
git cat-file --batch-check --batch-all-objects
# With MIDX: 2x faster than scanning all idx files

Performance Tuning

Key Git configs for large repositories: core.preloadIndex=true (parallel index loading), core.fsmonitor (watchman for file system changes), core.untrackedCache=true (cache untracked file scan), core.commitGraph=true, feature.manyFiles=true (enables all large-repo optimizations), protocol.version=2 (faster negotiation). Use Scalar for Microsoft's enhanced Git client.

language-bash
# Performance tuning for large repos
git config feature.manyFiles true
# Enables: commitGraph, fsmonitor, indexVersion >= 4, untrackedCache

# Enable protocol v2 for faster fetch negotiation
git config protocol.version 2

# Preload index for faster status
git config core.preloadIndex true

# Use file system monitor (watchman required)
git config core.fsmonitor true
# Falls back to: git config core.fsmonitor none

# Set delta cache size
git config core.deltaBaseCacheLimit 2g

# Enable sparse index
git sparse-checkout init --cone --sparse-index

# Install Scalar (Microsoft)
# brew install microsoft/git/scalar
scalar register
scalar clone <url> --single-branch

Debugging Tools

Git's internal debugging tools: GIT_TRACE=1 (trace git commands), GIT_TRACE_PACKET=1 (trace network protocol), GIT_TRACE_PERFORMANCE=1 (timing data), GIT_TRACE_SETUP=1 (path resolution). Use git rev-list --debug for walk details, git pack-objects --debug for packing decisions, and git fsck --strict for repo health.

language-bash
# Trace environment variables
export GIT_TRACE=1
export GIT_TRACE_PERFORMANCE=1
export GIT_TRACE_PACKET=1
export GIT_TRACE_SETUP=1

# Profile specific commands
GIT_TRACE_PERFORMANCE=2 git status

# Debug object lookup
git rev-list --debug --all --max-count=10

# Debug pack file contents
git verify-pack -v .git/objects/pack/pack-*.idx | grep -E "commit|blob|tree"

# Debug reachability
git rev-list --objects --all --use-bitmap-index --count

# Debug protocol negotiation
git -c protocol.version=2 fetch --verbose origin

# Analyze repository
git filter-repo --analyze

# Count objects by type
git cat-file --batch-check --batch-all-objects | awk '{print $2}' | sort | uniq -c | sort -rn
lightbulbDeep Dive & Key Takeawaysexpand_more

Understanding Git internals is essential for Staff+ engineers debugging performance issues or customizing Git for organizational needs. The commit-graph binary format stores: OID, parent offsets, root tree OID, generation number, commit timestamp. Generation numbers are the key innovation — they enable topological sorting without graph traversal and allow pruning during merge-base computation. Bitmaps use a two-level structure: a commit bitmap (one per ref) and a type bitmap (commits/trees/blobs/tags). MIDX files solve the 'many packfiles' problem — repos with 100+ packfiles see 5-10x query slowdown without MIDX. Microsoft's Scalar adds prefetch, background maintenance, and GVFS protocol for Windows. For deep debugging, build Git from source with DEVELOPER=1 for extra assertions and gdb breakpoints.
