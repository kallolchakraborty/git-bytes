Packages & Registries

schedule2 min readsignal_cellular_1_barFoundational
githubpackagesregistry

GitHub Packages is a unified package registry that supports multiple ecosystems — Container Registry (OCI-compliant), npm, Maven, NuGet, RubyGems, and more. Packages integrate natively with GitHub Actions, share permissions with repository access, and provide a single home for all your artifacts from build to distribution.

quick_reference Staff+ Cheat Sheetexpand_more

Core Commands

  • docker push ghcr.io/owner/repo:tag — push container to GHCR.
  • npm publish --registry=https://npm.pkg.github.com — publish npm package.
  • gh api /user/packages — check storage usage.

Staff+ Patterns

  • Use multi-arch builds (docker buildx build --platform linux/amd64,linux/arm64) — essential for mixed clusters.
  • Automate package pruning via API to control storage costs: DELETE /orgs/org/packages/container/repo/versions/$id.

Watch Outs

  • GHCR rate limit: 5000 pulls/hr authenticated, 100/hr anonymous. Use pull-through caching for CI.
  • Private packages count against storage quota; public packages are free. Don't accidentally expose internal images.

Interview Angle

Comes up in CI/CD system design (Netflix L6). Discuss artifact storage: blob store + metadata DB + CDN, and trade-offs between S3, S3+CloudFront, and OCI-compliant registries.

Package Ecosystems

GitHub Packages supports: Container Registry (GHCR, OCI-compliant, Docker/OCI images), npm (JavaScript/TypeScript, scoped packages), Maven (Java, Gradle), NuGet (.NET, symbols), RubyGems (Ruby), Cargo (Rust), and more. Each ecosystem uses its native client tooling with GitHub authentication.

Packages are scoped to a repository or organization. Public packages are free with unlimited storage and bandwidth. Private packages use the included storage/bandwidth limits depending on your plan.

Container Registry

GitHub Container Registry (GHCR) at ghcr.io stores Docker and OCI container images. It supports multi-architecture builds (linux/amd64, linux/arm64), OCI artifacts, and Cosign/Sigstore attestations for supply chain security.

Authenticate with echo $GITHUB_TOKEN | docker login ghcr.io -u USERNAME --password-stdin. Images are automatically built and pushed from GitHub Actions workflows with built-in caching and layer optimization.

Actions Integration

GitHub Packages integrates directly with Actions using the GITHUB_TOKEN for authentication — no additional credentials needed. Use docker/build-push-action for containers, actions/setup-node for npm, and ecosystem-specific publish actions for other registries.

Common patterns: publish on release tags, publish on push to main, publish for every commit (with versioning), and promote between staging and production registries using Actions environments.

Access Control

Package permissions are inherited from the repository: repository-scoped packages grant read/write/admin based on collaborator roles. For organization-scoped containers (GHCR), granular permissions control who can push and pull independently of repository access.

Fine-grained PATs can restrict tokens to specific package scopes and operations. SSO enforcement applies to package access within organizations. Internal packages are visible to all organization members but not to the public.

Storage & Bandwidth

GitHub Packages usage counts toward your account's storage and bandwidth limits. Free accounts: 500 MB storage / 1 GB bandwidth. Pro: 2 GB / 10 GB. Team: 2 GB / 10 GB. Enterprise: 50 GB / 100 GB. Additional storage and bandwidth can be purchased.

Public packages do not consume storage or bandwidth from your quota. Use gh api /user/packages to check usage, or visit Settings → Billing → Packages. Retention policies through Actions can automatically prune old package versions to manage costs.

lightbulbDeep Dive & Key Takeawaysexpand_more

GitHub Packages unifies artifact management across your entire software lifecycle. From CI/CD build artifacts to production container images, everything lives alongside your source code with consistent access control and audit trails.

Container Registry Architecture

GHCR is a fully OCI-compliant registry, meaning it handles not just container images but any OCI artifact — Helm charts, attestations, SBOMs, and WASM modules. Each image supports multiple tags, and ghcr.io/owner/repo/image:tag follows the standard Docker naming convention. Rate limits apply: 5000 pulls per hour per authenticated user (or 100/hour for anonymous).

npm Registry Configuration

To use GitHub Packages as an npm registry, configure .npmrc with @owner:registry=https://npm.pkg.github.com. Authentication requires a PAT with read:packages scope. For publishing, the PAT needs write:packages and the repository permission. GitHub Actions can use GITHUB_TOKEN automatically with actions/setup-node.

Maven & Gradle Configuration

GitHub Packages supports Apache Maven and Gradle. Configure settings.xml with the GitHub Packages server for Maven, or build.gradle repositories for Gradle. Authentication uses PAT with read:packages/write:packages. The repository URL pattern is https://maven.pkg.github.com/owner/repo.

Code Examples

# Authenticate Docker to GHCR
echo $GITHUB_TOKEN | docker login ghcr.io -u $GITHUB_ACTOR --password-stdin

# Build and push container image with multi-arch support
docker buildx build \n  --platform linux/amd64,linux/arm64 \n  -t ghcr.io/owner/repo:latest \n  -t ghcr.io/owner/repo:v1.2.3 \n  --push .

# Publish npm package from Actions
npm publish --registry=https://npm.pkg.github.com

# Configure .npmrc for GitHub Packages
echo "@owner:registry=https://npm.pkg.github.com" >> .npmrc
echo "//npm.pkg.github.com/:_authToken=$GITHUB_TOKEN" >> .npmrc

# NuGet configuration (nuget.config)
<?xml version="1.0" encoding="utf-8"?>
<configuration>
  <packageSources>
    <add key="github" value="https://nuget.pkg.github.com/owner/index.json" />
  </packageSources>
  <packageSourceCredentials>
    <github>
      <add key="Username" value="$GITHUB_ACTOR" />
      <add key="ClearTextPassword" value="$GITHUB_TOKEN" />
    </github>
  </packageSourceCredentials>
</configuration>

# List packages via API
gh api /user/packages
