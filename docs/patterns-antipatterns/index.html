Patterns & Anti-Patterns

schedule2 min readsignal_cellular_1_barFoundational
careerpatternsbest-practices

Recognize and remediate common Git anti-patterns while adopting battle-tested patterns used by high-performing engineering teams for cleaner history and smoother collaboration.

quick_reference Staff+ Cheat Sheetexpand_more

Core Commands

  • git add -p — build atomic commits from a messy working tree.
  • git rebase -i HEAD~N — clean up commits before sharing.
  • git filter-repo --path-glob '*.bin' --invert-paths — remove large files from history.
  • git push --force-with-lease — safe force-push after rebase.

Staff+ Patterns

  • Replace merge-commit workflow with rebase workflow — eliminates merge bubbles.
  • Adopt commitlint + husky client-side hooks to enforce conventions before push.
  • Use GitHub branch protection: require PR review, status checks, up-to-date branches.
  • Automate stale branch cleanup with a weekly GitHub Action.

Watch Outs

  • Big Bang commits (50+ files) — impossible to review, risky to revert. Use git add -p.
  • Frequent git merge main in feature branches creates messy history. Use rebase.
  • Late push before deadline always creates merge nightmare — push at least daily.

Interview Angle

Netflix L6: "How do you detect and remediate anti-patterns across 200 repos?" Use gh api repos/$ORG/$REPO/branches to audit branch protection compliance. Build a dashboard showing which repos lack protection rules, have stale branches, or show "merge commit" patterns in their history.

Common Patterns

Atomic Commits: each commit represents one logical change — easier to review, revert, and cherry-pick. Frequent Push: push at least daily to prevent data loss. Rebase Before Merging: keep feature branches up to date with main via rebase, not merge. Descriptive Messages: conventional commits with bodies explaining motivation. Feature Flags: decouple deployment from release — merge incomplete features behind flags.

language-bash
# Pattern: Atomic commits
git add -p  # interactively stage hunks
# Stage only related changes, commit, repeat

# Pattern: Frequent rebase to keep branch current
git checkout feature
git fetch origin
git rebase origin/main
# Resolve conflicts if any, continue
# Force push to update PR (with --force-with-lease)
git push --force-with-lease origin feature

# Pattern: Feature flags
if feature_flags.enabled("new-checkout"):
    render_new_checkout()
else:
    render_old_checkout()

Anti-Patterns to Avoid

Big Bang Commits: massive commits touching 50+ files — impossible to review, risky to revert. Merge Commits in Feature Branches: frequent git merge main creates messy history; use rebase instead. Committing Configuration: .env, IDE settings, local configs in .gitignore. Broken Main: merging untested code breaks everyone — enforce CI gates. Late Push: pushing right before deadline creates merge nightmare.

language-bash
# Anti-pattern: Big bang commit (DON'T)
git add -A && git commit -m "lots of changes"
# Instead: use interactive staging and commit per concern

# Anti-pattern: Merging main repeatedly (DON'T)
git checkout feature
git merge main  # creates ugly merge bubbles
# Instead: rebase
git rebase main

# Anti-pattern: Committing secrets (DON'T)
echo "API_KEY=abc123" >> .env
git add .env && git commit -m "add config"
# Always check: git add -p, .gitignore, pre-commit hooks

Migration Strategies

Migrating from anti-patterns to best practices: History Rewriting: use git filter-repo to remove large files, secrets, or PII. Rebranching: create clean branches from a sanitized point. Gradual Adoption: introduce conventional commits via commitlint; enforce with CI. Training: pair programming and PR reviews to teach patterns. Automation: pre-commit hooks for linting, secret scanning, and formatting.

language-bash
# Migration: Remove large blobs from history
git filter-repo \
  --path-glob '*.mp4' \
  --path-glob '*.zip' \
  --invert-paths

# Migration: Enforce conventional commits
# .commitlintrc.json
{
  "extends": ["@commitlint/config-conventional"],
  "rules": {
    "type-enum": [2, "always", ["feat", "fix", "docs", "chore", "refactor", "test", "perf"]]
  }
}

# Migration: Add pre-commit hooks
# .pre-commit-config.yaml
repos:
  - repo: https://github.com/pre-commit/pre-commit-hooks
    rev: v4.5.0
    hooks:
      - id: trailing-whitespace
      - id: end-of-file-fixer

Team Conventions

Document a team Git playbook: Branch naming: feat/, fix/, chore/, docs/, refactor/, test/. PR size: max 400 lines changed. Review turnaround: <24h for review. Merge strategy: squash-merge or rebase-merge, never merge-commit. Release process: tagged releases with release notes. Emergency process: revert button first, fix later.

language-bash
# Team Git playbook template

## Branch naming
feat/<short-description>    # new features
fix/<short-description>     # bug fixes
chore/<short-description>   # maintenance
docs/<short-description>    # documentation
refactor/<short-description> # code restructuring

## PR best practices
- Max 400 lines changed per PR
- Include test coverage
- Reference issue: "Closes #123"
- Description: what + why
- Screenshots for UI changes

## Merge strategy
- Trunk-based repos: squash-merge
- Staged releases: rebase-merge
- Never use default merge commit

Automation

Automate best practices enforcement: CI gating: build + test + lint + type check must pass. Branch protection: require PR review, status checks, up-to-date branch. Auto-merge: for low-risk changes meeting all checks. Housekeeping bot: stale branch cleanup, merge conflict notifications, label management. Dependabot: automated dependency updates with PRs.

language-bash
# GitHub branch protection rules (via API or UI)
# Required:
# - Require pull request review (1 approval)
# - Require status checks (CI, lint, test)
# - Require branches to be up to date
# - Include administrators

# Stale branch cleanup (GitHub Action)
name: Stale Branches
on:
  schedule:
    - cron: '0 6 * * 1'  # every Monday
jobs:
  cleanup:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: |
          for branch in $(git branch -r --merged origin/main); do
            echo "Delete stale: $branch"
            git push origin --delete ${branch#origin/}
          done
lightbulbDeep Dive & Key Takeawaysexpand_more

Teams that adopt Git best practices see 40% fewer merge conflicts and 60% faster PR cycle times. The most impactful anti-pattern to eliminate is the 'merge tsunami' — large, infrequent merges that create conflict hell. Replace with trunk-based development or frequent rebasing. The second most common issue is history pollution from WIP commits, large files, and secrets — solve with pre-commit hooks and git add -p habit. At scale, invest in automation: merge queues, CI gating, and automated refactoring tools like large-repo-tools and git-filter-repo.
