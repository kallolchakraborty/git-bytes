GitHub CLI (gh)

schedule2 min readsignal_cellular_1_barFoundational
githubcliautomation

The GitHub CLI (gh) brings GitHub workflows to your terminal — manage issues, PRs, releases, Actions, and more without leaving the command line. It wraps both REST and GraphQL APIs under a unified interface, supports extensions for custom commands, and integrates natively with CI/CD pipelines.

quick_reference Staff+ Cheat Sheetexpand_more

Core Commands

  • gh pr create --fill --reviewer @me — create PR from current branch.
  • gh pr merge --squash — squash merge.
  • gh issue list --label bug --json number,title — structured JSON output.
  • gh release create v1.0 --generate-notes — release with auto-release notes.
  • gh api /repos/{owner}/{repo} — raw API access.

Staff+ Patterns

  • Use gh api graphql -f query=... for complex queries beyond built-in commands — makes gh a universal GitHub client.
  • Script with --json --jq '.[].title' for pipeline-friendly output parsing.

Watch Outs

  • GITHUB_TOKEN on CI is repo-scoped. For cross-repo operations, use a PAT with broader scope.
  • gh auth login stores creds in keyring. On headless CI, use GH_TOKEN env var.

Interview Angle

Proficiency with gh signals operational maturity. Mention gh api for automation at scale — shows understanding of the underlying API and its rate limits.

Getting Started

Install gh via brew install gh (macOS), winget install GitHub.cli (Windows), or the official package managers. Authenticate with gh auth login — supports interactive OAuth, token-based login, and machine-to-machine authentication for CI/CD.

Key environment variables: GH_TOKEN for authentication in non-interactive mode, GH_HOST for GitHub Enterprise Server, and GH_REPO to set the working repository.

Core Commands

Core commands map to GitHub workflows: gh pr create with --fill auto-populates title/body from commit messages, gh pr review --approve streamlines code review, gh issue list --label bug filters issues, gh release create v1.0 automates releases, and gh run watch tails Actions logs in real-time.

The gh search command searches repos, issues, PRs, and code directly from the terminal with GitHub's search syntax.

gh extensions

Extensions add custom commands to gh. Browse the marketplace at gh extension list or install with gh extension install <repo>. Popular extensions: gh copilot (AI assistant), gh browser (open in browser), gh attestation (Sigstore signing), and gh project (Projects v2 management).

Create your own extension with gh extension create — extensions are executable scripts named gh-<name> in any language. Share via GitHub repos with the gh-extension topic.

Scripting with gh

gh is designed for scripting. Use --json flag for structured output, --jq for JSON filtering with jq syntax, and --template for Go template rendering. Combine with xargs, jq, and shell loops for batch operations.

The gh api command exposes raw REST and GraphQL endpoints when built-in commands don't cover a use case.

CI/CD Integration

gh is pre-installed on GitHub Actions runners. Use ${{ secrets.GITHUB_TOKEN }} for automatic authentication in workflows. Common CI patterns: gh pr merge --squash in auto-merge workflows, gh release upload for artifact publishing, and gh run rerun for flaky test retries.

For self-hosted runners, install gh and authenticate via GH_TOKEN environment variable with a fine-grained PAT.

lightbulbDeep Dive & Key Takeawaysexpand_more

The GitHub CLI is the definitive terminal interface for GitHub. It's maintained by the GitHub organization, receives monthly releases, and is designed to be composable — every command outputs structured JSON for piping into other tools.

Authentication Architecture

gh auth login creates a local OAuth session stored in the system keyring. For CI/CD and automation, set GH_TOKEN or GITHUB_TOKEN environment variables — gh detects these automatically. Enterprise Server users set GH_HOST to their instance URL. Use gh auth status to verify active sessions.

JSON Output & Filtering

The --json flag combined with --jq enables powerful data extraction. For example, gh pr view 42 --json title,body,reviews --jq '.title' extracts just the PR title. The --template flag uses Go template syntax for custom formatting.

Git Integration

gh integrates deeply with Git — gh pr create pushes the current branch if needed, gh repo clone defaults to SSH, and gh pr checkout fetches and checks out PR branches. Use gh browse to open the current repository's GitHub page in your browser.

Code Examples

# Authenticate for CI/CD
export GH_TOKEN="ghp_xxxxxxxxxxxx"

# Create PR with auto-fill and assign reviewers
gh pr create --fill --reviewer team-leads

# List open issues with label filter as JSON
gh issue list --label "bug" --json number,title,labels,assignees

# Batch close all issues with a specific label
gh issue list --label "stale" --json number \n  | jq -r '.[].number' \n  | xargs -I {} gh issue close {} --comment "Closing stale issue"

# Run a custom GraphQL query
gh api graphql -f query='
  query($owner:String!,$repo:String!){
    repository(owner:$owner,name:$repo) {
      pullRequests(first:5,states:[OPEN]) {
        edges { node { title url } }
      }
    }
  }' -F owner=github -F repo=docs

# Create a release and upload artifacts
gh release create v1.0.0 --title "v1.0.0" --notes "Release notes" ./dist/*.tar.gz

# Watch workflow runs for current branch
gh run watch --exit-status
