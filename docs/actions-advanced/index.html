GitHub Actions Advanced

schedule2 min readsignal_cellular_4_barAdvanced / Staff+
devopsactionsadvancedstaff+

Advanced GitHub Actions patterns including reusable workflows, composite actions, build matrices, and complex orchestration strategies for enterprise-scale CI/CD pipelines.

quick_reference Staff+ Cheat Sheetexpand_more

Core Commands

  • workflow_call — reusable workflows, called from other workflows.
  • workflow_run — trigger workflows based on another workflow's completion.
  • environment: production — deployment environments with approvals.
  • concurrency: group: ci-${{ github.ref }} — cancel duplicates.

Staff+ Patterns

  • Use reusable workflows with workflow_call to define a deployment pipeline once and share across all repos — DRY your CI/CD.
  • Use matrix with include for conditional job variants: matrix: os: [ubuntu, macos], include: [{ os: ubuntu, node: 18 }, { os: macos, node: 20 }].

Watch Outs

  • Reusable workflows can't call other reusable workflows (no nesting). Use composite actions instead for nested reuse.
  • GITHUB_TOKEN in reusable workflows is the caller's token, not the callee's. Permission issues arise when the caller doesn't have access to the callee's secrets.

Interview Angle

Netflix L7 asks about deployment pipeline design at scale. Discuss the trade-off between workflow_run (decoupled, async) and needs (coupled, synchronous). The key insight is idempotency and rollback strategy.

Build Matrices

Generate combinatoric job variants using strategy.matrix to test across OS, language versions, and configuration combinations efficiently.

language-yaml
jobs:
  test:
    strategy:
      matrix:
        os: [ubuntu-latest, windows-latest, macos-latest]
        node: [18, 20, 22]
        include:
          - os: ubuntu-latest
            node: 22
            experimental: false
        exclude:
          - os: macos-latest
            node: 18
    runs-on: ${{ matrix.os }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm ci && npm test

Reusable Workflows

Call workflows from other workflows using workflow_call trigger. Supports input/output contracts and secret inheritance for DRY pipeline composition.

language-yaml
# caller.yml
jobs:
  call-workflow:
    uses: ./.github/workflows/reusable-ci.yml
    with:
      node-version: 20
      deploy: ${{ github.ref == 'refs/heads/main' }}
    secrets: inherit

# reusable-ci.yml
on:
  workflow_call:
    inputs:
      node-version:
        type: number
        required: true
      deploy:
        type: boolean
        default: false
    secrets:
      npm-token:
        required: true
    outputs:
      build-id:
        description: "Build identifier"
        value: ${{ jobs.build.outputs.id }}

Composite Actions

Bundle multiple steps into a reusable action using action.yaml. Composites wrap shell commands and other actions without requiring a Docker container or JavaScript runtime.

language-yaml
# .github/actions/setup-project/action.yaml
name: "Setup Project"
inputs:
  node-version:
    required: true
    default: 20
runs:
  using: "composite"
  steps:
    - uses: actions/checkout@v4
    - uses: actions/setup-node@v4
      with:
        node-version: ${{ inputs.node-version }}
        cache: npm
    - run: npm ci
      shell: bash
    - run: npm run build
      shell: bash

Docker & JS Actions

JavaScript actions (action.yml with node20) run directly on the runner. Docker actions containerize the environment for reproducible execution with pinned dependencies.

language-yaml
# JavaScript action (action.yml)
name: "My JS Action"
runs:
  using: "node20"
  main: "dist/index.js"

# Docker action (action.yml)
name: "My Docker Action"
runs:
  using: "docker"
  image: "Dockerfile"
  env:
    INPUT_DIR: /workspace
  args:
    - ${{ inputs.path }}

Complex Orchestration

Chain jobs with needs, use concurrency groups for environment safety, implement required and optional checks, and merge multiple workflow results.

language-yaml
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - run: npm run lint

  build:
    needs: lint
    strategy:
      matrix:
        arch: [x64, arm64]
    runs-on: ${{ matrix.arch == 'arm64' && 'arm-runner' || 'ubuntu-latest' }}
    steps:
      - run: npm run build

  deploy:
    if: github.ref == 'refs/heads/main'
    needs: [build]
    concurrency:
      group: environment-prod
      cancel-in-progress: false
    environment: production
    runs-on: ubuntu-latest
    steps:
      - run: npm run deploy
lightbulbDeep Dive & Key Takeawaysexpand_more

GitHub Actions advanced patterns enable enterprise teams to build maintainable, scalable CI/CD pipelines. Build matrices reduce boilerplate by generating job variants from parameter combinations. Reusable workflows (workflow_call) promote DRY principles across repositories with typed inputs, outputs, and secret inheritance. Composite actions encapsulate step sequences without external packaging — ideal for team-specific setup routines. Docker actions provide fully hermetic environments, while JavaScript actions offer the fastest startup for tooling built with the GitHub Actions Toolkit. For complex orchestration, combine needs with concurrency groups and environment protection rules to model safe deployment pipelines with manual approval gates at critical transition points.
