Security & Dependabot

schedule2 min readsignal_cellular_1_barFoundational
githubsecuritydependabot

GitHub's security suite protects your software supply chain from dependency vulnerabilities, leaked secrets, and malicious code. Dependabot automates dependency updates, secret scanning detects credentials in code, CodeQL performs semantic code analysis, and security advisories coordinate vulnerability disclosures — all working together to secure every stage of development.

quick_reference Staff+ Cheat Sheetexpand_more

Core Commands

  • .github/dependabot.yml — config for automated dependency updates.
  • github/codeql-action@v3 — CodeQL analysis in CI.
  • GH_TOKEN with security_events: write — API-based security management.

Staff+ Patterns

  • Dependabot with open-pull-requests-limit: 10 + reviewers: team-security. Auto-merge patches, manually review major updates.
  • Use CodeQL security-extended query suite (not just default) — catches XSS, SQL injection, path traversal.

Watch Outs

  • Secret scanning push protection can block legitimate commits. Have a bypass process (admins can allow specific pushes).
  • Dependabot PRs consume CI minutes. Set schedule.interval: weekly to avoid daily CI storms.

Interview Angle

Stripe/Microsoft ask about supply chain security. The key insight is defense-in-depth: Dependabot + CodeQL + secret scanning + signed commits. Discuss the friction vs security trade-off.

Dependabot Alerts

Dependabot alerts notify you when repositories depend on packages with known security vulnerabilities. Alerts are triggered by the GitHub Advisory Database — a curated list of CVEs and GitHub-reviewed advisories. Each alert includes severity (Critical/High/Medium/Low), CVE identifier, affected version range, and fixed version.

Enable alerts in repository settings → Security & analysis → Dependabot alerts. For organizations, enable at the org level to cover all repos. Alerts appear in the Security tab and can be routed to Slack, email, or Teams via webhook integration.

Dependabot Updates

Dependabot automates the remediation of vulnerable dependencies by creating pull requests that bump the dependency to the patched version. Configuration lives in .github/dependabot.yml and supports multiple package ecosystems in a single file.

Key configuration options: schedule.interval (daily, weekly, monthly), open-pull-requests-limit (max concurrent PRs), labels for automated triage, reviewers for mandatory code review, and ignore rules to skip specific dependencies or updates.

Secret Scanning

Secret scanning automatically detects secrets — API keys, tokens, private keys, and passwords — pushed to repositories. GitHub maintains patterns for over 200+ service providers (AWS, Azure, GCP, Stripe, Slack, etc.) and supports custom patterns for organization-specific secrets.

When a secret is detected, GitHub alerts the repository admin and the secret's service provider (if partnered). Push protection blocks commits containing supported secrets, preventing leaks before they happen.

CodeQL Integration

CodeQL is GitHub's semantic code analysis engine. It treats code as data — queries analyze the codebase for vulnerabilities like SQL injection, XSS, path traversal, and hardcoded credentials. CodeQL supports C/C++, C#, Go, Java, JavaScript/TypeScript, Python, and Ruby.

Enable CodeQL via GitHub Actions with the github/codeql-action — it autobuilds supported languages and runs default query suites (security-extended, security-and-quality). Results appear as code scanning alerts in the Security tab with suggested fixes.

Security Advisories

GitHub Security Advisories provide a private, collaborative space to draft, discuss, and publish vulnerability disclosures. Advisory drafts include CVE request, affected versions, severity, CVSS score, and patched versions. Once published, the advisory feeds into the GitHub Advisory Database and triggers Dependabot alerts for affected repositories.

Temporary private forks let collaborators work on a fix before public disclosure — the fork is deleted once the advisory is published. Advisories support credit attribution to security researchers who reported the vulnerability.

lightbulbDeep Dive & Key Takeawaysexpand_more

GitHub's security features form a layered defense for the software supply chain. Dependabot handles dependency health, secret scanning prevents credential leaks, CodeQL catches code-level vulnerabilities, and security advisories coordinate responsible disclosure.

Dependabot Configuration Deep Dive

The dependabot.yml file supports multiple package ecosystems in a single configuration. Each ecosystem entry specifies the package manager, directory, update schedule, and customization options. Version update strategies include lockfile-only (safe, minimal), increase (bump to latest), and increase-if-necessary (bump the requirement only if needed by the lockfile).

Secret Scanning Partner Program

GitHub partners with over 200 service providers. When a secret is detected, GitHub sends an alert to both the repo admin and the provider, who can automatically revoke the exposed credential. Custom patterns use regex with optional file path matching for organization-proprietary secret formats.

CodeQL Query Customization

Beyond the default query suites, teams can write custom QL queries for organization-specific vulnerability patterns. The codeql CLI supports local query development and testing before deployment to CI/CD. Custom queries are reusable across repositories via shared GitHub Actions workflows.

Code Examples

# .github/dependabot.yml — Multi-ecosystem configuration
version: 2
updates:
  - package-ecosystem: "npm"
    directory: "/"
    schedule:
      interval: "weekly"
      day: "monday"
      time: "09:00"
      timezone: "America/New_York"
    open-pull-requests-limit: 10
    labels:
      - "dependencies"
      - "auto-merge"
    reviewers:
      - "team-security"
    ignore:
      - dependency-name: "lodash"
        update-types: ["version-update:semver-patch"]

  - package-ecosystem: "docker"
    directory: "/"
    schedule:
      interval: "weekly"
    labels:
      - "docker"
      - "dependencies"

# CodeQL workflow ( .github/workflows/codeql.yml )
name: "CodeQL"
on:
  push: { branches: [main] }
  pull_request: { branches: [main] }
jobs:
  analyze:
    name: Analyze
    runs-on: ubuntu-latest
    strategy:
      matrix:
        language: [javascript, python]
    steps:
      - uses: actions/checkout@v4
      - uses: github/codeql-action/init@v3
        with:
          languages: ${{ matrix.language }}
          queries: security-extended
      - uses: github/codeql-action/autobuild@v3
      - uses: github/codeql-action/analyze@v3
